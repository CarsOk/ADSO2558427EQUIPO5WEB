Estos diagramas se utilizan para modelar y visualizar la estructura, el comportamiento y las relaciones de los sistemas de software antes de implementarlos. Ayudan a diseñar y planificar el software antes de comenzar la codificación y pueden representar la estructura de clases, las relaciones entre objetos, el flujo de datos, la arquitectura del sistema y más; además sirven como documentación detallada del diseño del software que nos facilitan la comprensión de cómo funciona el sistema y cómo están organizados sus componentes.
